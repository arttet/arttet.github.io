# RFC Index <Badge type="tip" text="v1" />

Proposals for significant architectural changes. Open RFCs are discussed via PR; accepted RFCs become ADRs.

## Process

1. Copy `template.md` to a new file (e.g. `0001-my-new-feature.md`).
2. Fill out problem, proposed solution, alternatives, impact, open questions.
3. Open a Pull Request.
4. Discuss with reviewers.
5. If approved, the RFC summary moves into an ADR (currently paused) and implementation begins. The architecture spec under `docs/platform/architecture/` is updated in the same PR as the code.

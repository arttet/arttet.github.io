# Evolution <Badge type="warning" text="beta" />

The platform's evolution is itself a product. Every architectural change moves through a structured flow from raw idea to historical record, so future engineers (and AI agents) can reconstruct _why_ the system looks the way it does.

## Sections

- **[Workflow](./workflow.md)** — how an idea becomes an RFC, then an ADR, then a spec update.
- **[RFCs](./rfc/index.md)** — proposals open for discussion.
- **[ADRs](./adr/index.md)** — accepted decisions, append-only history.

## Status

The current codebase is `dev`. New ADRs are paused until the v1 cut. RFCs and the workflow itself remain active so proposals can still land in queue.
